什么？写了五年Java代码都没自己编译过虚拟机？

2022-01-09
#虚拟机
code

最近闲着无事，又翻开了周志明老师的《深入理解Java虚拟机》这本书。已经不知道是第多少次翻开这本书了，似乎每次读都有不一样的理解。不得不说，有些书是常读常新。

回忆起从业两年刚拿到这本书的时候，大言不惭地放下大话：我一个月就能读完它。而事实上，也就勉强读完了讲虚拟机前世今生的历史故事第一章。当看到要动手编译一个JVM的时候，毫无悬念的被劝退。而如今好几年过去了，再看到这个部分的时候，也倒不觉得那么困难。

环境准备

OS : MacOS Big Sur 11.2

BOOT JDK : jdk-11.0.13+8

开发工具: Clion ,Clang(mac自带)

目标JDK源码：http://hg.openjdk.java.net/jdk-updates/jdk11u/file/ebc6c83bfa9e

首先是操作系统，建议使用类Unix系统来编译。windows上操作起来会增加很多难度，没必要折腾。

编译的目标jdk需要依赖现有的jdk，这里安装了一个openjdk的版本，作为用来编译目标jdk的环境。需要将环境变量设置为openjdk的，这里不作赘述。

虚拟机是用C/C++写的，如果要去读一读虚拟机的代码，Clion是一个不错的选择。而编译C/C++代码一般用我们熟知的GCC编译，Clang是一个稍微轻量级的编译器，同样也能编译C/C++代码。在MacOs中Clang系统自带，无需额外安装。

熟悉C/C++或者自己在unix上通过源码安装过软件的小伙伴都知道在使用make命令之前都需要执行一个config脚本。编译虚拟机也同样如此。

编译

编译其实十分简单，只需要一个命令就够了。

这个过程实际上还是挺费时间的。敲完命令机器负载狂飙。

验证

千呼万唤始出来，看自己编译的jdk能不能跑，最简单的办法是看看编译出来的java二进制文件能不能跑。切换到bin路径下使用命令java -version来观察输出。

和自己之前安装的java并不一个java版本，很棒，it works.

调试

调试一下JDK听起来有点大言不惭，但来都来了，留下点记号呗。

先来点简单的，jvm启动的时候输出点自己的【私货】。

首先需要配置一下Clion，前提是先导入openjdk源码文件夹下的src到Clion中，然后再配置一下刚才编译好的执行程序。
在源码中打断点，就能对jvm进行调试了。

配置Clion只是编译的一个快捷方式，其实相当于使用make命令。改了源码需要重新编译一次，不必使用make clean，否则又得等上20分钟。

在源码的启动入口加点自己的输出。


至此，一个属于自己的jvm虚拟机就编译好了。

总结

  • 编译jvm一点都不麻烦，有手就行
  • 随着屏幕上的字符按照自己预期那样跳动，眼前的一切都变得索然无味
  • 如果一个java程序员没编译过jvm，那就不算一个真正的javer
